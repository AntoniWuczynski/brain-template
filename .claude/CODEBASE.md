# CODEBASE.md — vault internals for Claude Code

A short orientation document for Claude when working in this repo. Other
agents should read [`../AGENTS.md`](../AGENTS.md) first; this file
focuses on Claude-specific navigation aids and is **not** a substitute
for the rules there.

## Where things live

- **Ground-truth files**: `archive/raw/<topic>/<file>` — never edit.
- **Extracted artefacts**: `archive/processed/<topic>/<file>.md` plus
  any `_assets/` directory next to it for figures/tables.
- **Index notes**: `knowledge/index/<topic>/<file>.md` — the
  Obsidian-readable counterparts to processed artefacts. These are
  generated by ingestion but may be hand-edited; user edits to
  frontmatter are preserved on re-ingest.
- **Curated notes**: anywhere else under `knowledge/`. These are
  hand-written and outrank generated content.
- **State**: `metadata/index.jsonl` (one JSON object per file) and
  `logs/ingest-*.log`.
- **Tooling**: `scripts/ingest.py` (CLI) and `scripts/ingest_lib/`
  (extractors, hashing, metadata I/O, note generation).

## Where Claude should look first

| Question | First place to look |
|---|---|
| Has file X been processed? | `metadata/index.jsonl`, grep by `relative_path` or `source_hash`. |
| What's in lecture Y? | `archive/processed/<rel>.md` first, then the source. |
| What does the agent contract say? | `../AGENTS.md`. |
| Why did the last run skip Z? | `logs/ingest-<latest>.log`. |
| How is extension `.foo` handled? | `scripts/ingest_lib/extractors/__init__.py` registry. |

## Patterns to follow

- Treat `metadata/index.jsonl` as append-mostly. To "update" a record,
  append a new line and let downstream tooling pick the latest by
  `(relative_path, source_hash)` keying.
- Wikilinks use repo-relative paths without `.md`:
  `[[archive/raw/university/COMP0101/01_foundations]]`.
- When generating Markdown that other agents will read, prefer plain
  ATX headings (`# foo`), no HTML, no fancy tables.

## Things Claude should not do here

- Keep CI lean, don't gold-plate it. The framework here syncs to the
  **public** `brain-template` repo, which needs testing, so a single
  `.github/workflows/ci.yml` (pytest + ruff + mypy on one Ubuntu / Python
  3.12 runner, skipping `mcp(...)` vault-note commits) is expected and must
  stay green. What to avoid is the over-built kind: version/OS matrices,
  coverage gates, packaging/wheels.
- Don't reach for ML to "summarise" failed extractions. Mark them and
  surface them to the user.
- Don't drift the MCP server from its contract. The server IS
  implemented in `mcp_server/`; treat `mcp/README.md` as its contract
  and keep code and contract in sync.

## Claude memory and patterns

  this specific repo's quirks. Add only when surprising or non-obvious.
- `.claude/patterns/` — example snippets / templates referenced from
